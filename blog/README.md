# How to Write a Blog Post

This website uses GitHub Pages' built-in Jekyll support. You do not need to install anything locally.

Each blog post is a Markdown file in the `_posts/` folder.

## Add a new post

1. Duplicate `blog/template.md`.
2. Move the copy into `_posts/`.
3. Rename the copy with this format: `YYYY-MM-DD-short-title.md`.
   - Example: `2026-05-02-my-new-note.md`.
3. Open the new file in a text editor.
4. Replace:
   - `POST TITLE`
   - `Short description of this post.`
   - `CATEGORY`
   - the writing below the second `---` line.
5. Commit and push the new Markdown file.
6. GitHub Pages will automatically add it to `https://decoderliu.github.io/blog/`.

## Rules of thumb

- Keep file names simple: lowercase letters, numbers, and hyphens.
- Do not rename existing files after publishing unless you also update links.
- Use blank lines between paragraphs.
- Use `##` for section headings.
- Use `-` for bullet lists.
- Use backticks for code-like text, for example `u_t`.
